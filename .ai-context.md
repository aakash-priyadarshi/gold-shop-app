## рџљ« NEVER DO THESE THINGS

### Database Operations
1. **NEVER run `prisma migrate dev`** - This can reset the production database!
2. **NEVER run `prisma db push --force-reset`**
3. **NEVER run any command that says "All data will be lost"**
4. **Always ASK the user before running ANY Prisma migration commands**

### Safe Migration Workflow
```bash
# For LOCAL development only (with local DB):
npx prisma migrate dev --name <migration_name>

# For PRODUCTION - ONLY use:
npx prisma migrate deploy  # Applies pending migrations without reset

# To generate client without migration:
npx prisma generate
```
