## MODULE PURPOSE

The Emotion Engine is the "secret sauce" that makes customers feel like they're
talking to an unusually perceptive, emotionally intelligent human — not a bot.

It operates on two levels simultaneously:

1. DETECTION — reads the customer's emotional state in real-time from voice + words
2. ADAPTATION — adjusts the AI's voice, tone, pacing, and language in response

This module touches every part of the call. It runs continuously in the background,
updating the AI's behavior every 2-3 seconds based on how the customer is feeling.

The goal: customer hangs up thinking "that person really understood me."

---

## CORE PRINCIPLE

```
We are not detecting emotions to label them.
We are detecting emotions to RESPOND to them appropriately.

Every detection must produce an action.
Every action must serve one purpose: make the customer feel heard.
```

---

## ARCHITECTURE OVERVIEW

```
INPUTS (parallel, continuous):
  ├── Raw audio stream (Hume AI voice emotion analysis)
  ├── Real-time transcript (word-level sentiment)
  ├── Speaking patterns (speed, volume, pause frequency)
  └── Conversation history (context-aware emotion tracking)

PROCESSING:
  ├── EmotionDetector → EmotionState (every 2s)
  ├── EmotionHistory → trend analysis (rising? falling? stable?)
  ├── FrustrationMonitor → threshold alerts
  └── BuyingSignalDetector → escalation triggers

OUTPUTS (what changes based on emotion):
  ├── ToneConfig → ElevenLabs voice parameters
  ├── PacingConfig → speaking speed, pause duration
  ├── FillerContext → which filler type to use
  ├── ResponseStrategy → how to frame next message
  ├── EscalationTrigger → hand off to human if needed
  └── CRMEmotionLog → emotional arc saved to database
```

## TYPE DEFINITIONS

```typescript
// The primary emotional state detected from customer voice + words
interface EmotionState {
  // Primary detected emotion
  primary: EmotionLabel;

  // Secondary emotion (humans feel multiple things at once)
  secondary?: EmotionLabel;

  // How strongly the emotion is expressed (0 = barely detectable, 1 = very strong)
  intensity: number; // 0-1

  // How confident we are in this detection (below 0.5 = treat as 'neutral')
  confidence: number; // 0-1

  // Raw voice metrics that fed into this detection
  voiceMetrics: VoiceMetrics;

  // Timestamp of this snapshot
  detectedAt: Date;

  // Which data sources contributed to this detection
  sources: DetectionSource[];
}

type EmotionLabel =
  | "excited" // Speaking fast, higher pitch, positive words
  | "interested" // Leaning in, asking questions, engaged pace
  | "neutral" // Baseline — no strong signal either way
  | "hesitant" // Slow, quieter, lots of "um", "I'm not sure"
  | "skeptical" // Challenging tone, "but", "how do you know", "really?"
  | "frustrated" // Volume rising, sharp words, shorter sentences
  | "angry" // Loud, fast, cutting words, "always", "never"
  | "tired" // Slow, flat, minimal energy, short answers
  | "confused" // "Wait", "what do you mean", rising intonation on statements
  | "happy" // Laughing, warm tone, expansive responses
  | "distracted" // Background noise, answering own questions, trailing off
  | "considering" // Long pauses, "hmm", slower pace (positive — they're thinking)
  | "buying"; // Asking about next steps, pricing, timeline, onboarding

interface VoiceMetrics {
  wordsPerMinute: number; // Normal range: 120-180. Fast: >200, Slow: <100
  volumeLevel: number; // 0-1 normalized from audio amplitude
  volumeDelta: number; // Change from call baseline (+ = louder, - = softer)
  pitchMean: number; // Average Hz this turn
  pitchVariance: number; // Low variance = monotone/flat, High = expressive
  pauseFrequency: number; // Pauses per minute
  sentenceLengthAvg: number; // Short sentences = direct/frustrated. Long = engaged
  fillerWordRate: number; // "um", "uh", "like" per minute → hesitation signal
  questionRate: number; // Questions per minute → interest or confusion
}

type DetectionSource =
  | "hume_ai" // Hume AI voice emotion API
  | "transcript" // Keyword/phrase sentiment analysis
  | "voice_metrics" // Speed, volume, pitch analysis
  | "history"; // Trend from previous states

// What the AI should do differently based on emotional state
interface AdaptationConfig {
  // Voice parameters → passed to ElevenLabs
  tone: ToneConfig;

  // Conversation behavior → passed to ConversationBrain
  strategy: ResponseStrategy;

  // Whether to activate special modes
  activateSoftenMode: boolean;
  activateHighEnergyMode: boolean;
  activateSilenceMode: boolean; // Just listen, don't push content

  // Filler selection hint for FillerAudioManager
  fillerContext: FillerContext;

  // Backchannel behavior adjustment
  backchannelFrequency: "none" | "low" | "normal" | "high";
}

interface ToneConfig {
  // ElevenLabs stability: lower = more expressive
  // Excited customer → lower stability to match their energy
  // Tired/frustrated → higher stability for calm, steady presence
  stability: number; // 0.2-0.8

  similarityBoost: number; // 0.6-0.9 (usually keep high)
  style: number; // 0.0-0.5 (higher = more stylistic variation)

  // Speaking speed multiplier applied to ElevenLabs output
  // 1.0 = natural pace, 1.15 = slightly faster, 0.85 = slower
  speedMultiplier: number; // 0.75-1.2

  // Volume adjustment from AI baseline
  volumeAdjustment: number; // -0.25 to +0.15

  // Descriptive warmth level for prompt injection
  warmth: "clinical" | "neutral" | "warm" | "very_warm" | "intimate";

  // Energy level for prompt injection
  energy: "very_low" | "low" | "medium" | "high" | "very_high";

  // Formality for prompt injection
  formality: "casual" | "conversational" | "professional" | "formal";
}

interface ResponseStrategy {
  // Overall approach for this response
  mode:
    | "normal" // Standard conversation
    | "soften" // Back off, validate, reset — frustration detected
    | "energize" // Match excitement — customer is pumped
    | "slow_down" // Simplify, one thing at a time — confusion detected
    | "close" // Move toward commitment — buying signals detected
    | "listen" // Ask one question and STOP — customer needs to talk
    | "recover"; // Call went sideways — rebuild trust from scratch

  // Maximum words in next AI response (shorter when customer is frustrated/tired)
  maxResponseWords: number;

  // Whether to ask a question at the end of response
  endWithQuestion: boolean;

  // Silence duration to hold after speaking (ms)
  // CRITICAL: After price reveal or closing question → use long silence
  trailingSilenceMs: number;

  // Whether to validate customer's feeling before any content
  openWithValidation: boolean;

  // Whether to completely drop pitch/sales content this turn
  dropSalesContent: boolean;
}
```

## CLASS DEFINITIONS

### EmotionDetector (main entry point)

```typescript
/**
 * Orchestrates all emotion detection sources and produces
 * a unified EmotionState every 2 seconds during an active call.
 *
 * Uses sensor fusion — combines Hume AI, transcript analysis, and
 * voice metrics. No single source is trusted alone.
 *
 * Confidence weighting:
 *   Hume AI voice analysis:     45% weight
 *   Voice metrics (speed/vol):  35% weight
 *   Transcript keywords:        20% weight
 */
class EmotionDetector {
  /**
   * Feed raw audio chunk for Hume AI analysis.
   * Called on every audio packet — non-blocking, batches internally.
   * Hume AI results are async — they arrive ~200ms after audio is sent.
   */
  feedAudio(chunk: Buffer, timestamp: number): void;

  /**
   * Feed latest transcript text for keyword sentiment analysis.
   * Called on every Deepgram partial and final transcript.
   */
  feedTranscript(text: string, isFinal: boolean): void;

  /**
   * Feed voice metrics computed from audio analysis.
   * Called every 2 seconds with aggregated metrics window.
   */
  feedVoiceMetrics(metrics: VoiceMetrics): void;

  /**
   * Get current fused emotion state.
   * Returns null if insufficient data (< 3 seconds of call).
   */
  getCurrentState(): EmotionState | null;

  /**
   * Subscribe to emotion state updates.
   * Callback fires every 2 seconds with latest fused state.
   */
  onStateUpdate(callback: (state: EmotionState) => void): void;

  /**
   * Subscribe to high-priority emotion alerts.
   * Fires immediately when frustration or buying signal thresholds crossed.
   * Do not wait for the 2-second polling cycle for these.
   */
  onAlert(callback: (alert: EmotionAlert) => void): void;
}
```

### EmotionHistory

```typescript
/**
 * Tracks the emotional arc of the entire call.
 *
 * Used for:
 * - Detecting trends (getting better or worse?)
 * - Post-call CRM reporting (emotional journey)
 * - Deciding when to attempt close (sustained positive trend)
 * - Detecting when call is failing (consistent negative trend)
 *
 * Stores one EmotionState snapshot every 30 seconds + every major shift.
 */
class EmotionHistory {
  /**
   * Add a new emotion state to history.
   * Automatically detects if this is a "major shift" and flags it.
   */
  record(state: EmotionState): void;

  /**
   * Get the emotional trend over the last N seconds.
   * Returns: 'improving' | 'declining' | 'stable' | 'volatile'
   */
  getTrend(windowSeconds: number): EmotionTrend;

  /**
   * Get the best moment in the call so far.
   * Used by CRM report: "Customer was most engaged at 4:32"
   */
  getPeakEngagementMoment(): { timestamp: Date; state: EmotionState };

  /**
   * Get full emotional arc for post-call CRM report.
   * Returns array of {timestamp, emotion, intensity} for the full call.
   */
  getFullArc(): EmotionArcPoint[];

  /**
   * Determine if conditions are right to attempt a close.
   * Conditions: sustained positive trend for 90+ seconds,
   *             no frustration in last 60s, interest or buying signal present
   */
  isReadyToClose(): boolean;

  /**
   * Determine if call recovery is needed.
   * Conditions: 3+ consecutive negative states, or frustration intensity > 0.7
   */
  needsRecovery(): boolean;
}
```

### FrustrationMonitor

```typescript
/**
 * Dedicated monitor for frustration and anger signals.
 * Operates independently from the main EmotionDetector with
 * lower latency — frustration must be caught and responded to
 * within ONE exchange, not after a 2-second polling cycle.
 *
 * When frustration is detected, AI must:
 * 1. Immediately drop voice volume 10%
 * 2. Slow speaking speed 15%
 * 3. Abandon current sales track completely
 * 4. Respond with pure validation — no content
 * 5. Ask ONE open question and then just listen
 */
class FrustrationMonitor {
  /**
   * Process latest voice metrics and transcript for frustration signals.
   * Called on every transcript update — must be synchronous and fast.
   */
  update(metrics: VoiceMetrics, transcript: string): FrustrationLevel;

  /**
   * Returns current frustration level.
   * 'none' | 'mild' | 'moderate' | 'high' | 'critical'
   *
   * Thresholds:
   * mild:     volume +10%, 1 trigger phrase
   * moderate: volume +20%, 2+ trigger phrases, or sharp sentence shortening
   * high:     volume +30%, anger words, sentence length < 5 words avg
   * critical: raised voice + multiple anger signals → escalate to human
   */
  getLevel(): FrustrationLevel;

  /**
   * Get appropriate soften mode response for current frustration level.
   * Returns from pre-written library — NOT generated by LLM.
   * Reason: LLM generation takes 400-700ms. Soften response must
   *         play within one exchange. Pre-written = instant.
   */
  getSoftenResponse(level: FrustrationLevel, context: CallContext): string;

  /**
   * Reset frustration tracking after successful de-escalation.
   * Called when emotion returns to neutral/positive for 30+ seconds.
   */
  reset(): void;
}
```

## SILENCE INTELLIGENCE RULES

The silence rules are as important as the speaking rules.
Silence at the right moment = trust. Silence at the wrong moment = awkwardness.

```typescript
const SILENCE_RULES = {
  // After asking a closing question — NEVER speak first
  // Wait until customer breaks silence, no matter how long
  after_closing_question: {
    minWaitMs: 3000,
    maxWaitMs: Infinity, // Wait as long as it takes
    gentlePromptAfterMs: 8000, // "What are you thinking?"
  },

  // After revealing price — one of the most critical silences in sales
  // The first person to speak after price loses negotiating power
  after_price_reveal: {
    minWaitMs: 4000,
    maxWaitMs: Infinity,
    gentlePromptAfterMs: 10000, // "What are your thoughts on that?"
  },

  // When customer says "let me think about it"
  // Don't rush to fill — validate and give space
  after_think_about_it: {
    minWaitMs: 2000,
    response: "Of course — take your time.",
    followUpWaitMs: 5000, // Then gently: "Is there anything specific holding you back?"
  },

  // Customer has gone quiet mid-sentence (< 1.5s)
  // This is NOT a turn-end — it's a thinking pause
  // Backchannel = invitation to continue
  mid_sentence_pause: {
    maxWaitMs: 1500,
    action: "backchannel", // Play soft "mhm" or "yeah"
  },

  // After making a key point
  // Let it land — don't undercut it by adding more words
  after_key_point: {
    minWaitMs: 600,
    maxWaitMs: 1200,
  },
} as const;
```

## WHAT COPILOT SHOULD NEVER DO IN THIS MODULE

- Never make the AI mirror frustration or anger — always respond with calm
- Never detect emotion and do nothing — every state must produce an adaptation
- Never fill silence during 'considering' state — it's the most valuable silence in sales
- Never call Hume AI synchronously — always async, never block the audio pipeline
- Never hard-code emotion thresholds — put them in config so they can be A/B tested
- Never log raw audio to database — only transcript + emotion labels + metrics
- Never escalate to human based on a single signal — require sustained pattern
- Never play backchannel during angry/critical frustration — don't interrupt
- Never use ElevenLabs stability above 0.75 — sounds robotic and flat
- Never trust transcript alone for emotion — always fuse with voice metrics
- Never apply sudden voice parameter changes — interpolate gradually over 3-5 seconds
  to avoid jarring transitions that sound unnatural
